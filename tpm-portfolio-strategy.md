# TPM/PM Portfolio Strategy — Research Findings

## Identity Framing
- Lead with TPM; email dev and AI are credibility multipliers, not competing identities
- Avoid "AI Enthusiast" label; use verb-led specifics: named models, shipped tools, honest failure modes
- One-line value prop format: "I [verb] [outcome] for [audience] by [unique angle]"

## Section Order (recruiter-optimized)
Hero → Selected Work → About → Experience → Writing → Skills → Contact
(Projects above About — evidence beats narrative when attention is short)

## Project Case Study Structure
Context → Problem → Action → Result → Reflection (~300–500 words)
Lead with one-line outcome containing a real metric

## AI Presentation Rules
- Never: "AI enthusiast", "exploring LLMs", "leveraging GPT-4 to drive impact"
- Always: specific verb + named model + honest scope + one stated failure mode
- Highest-leverage move: ship one AI tool that serves TPM or email dev workflow

## Stack Recommendation
Plain HTML/CSS/JS on GitHub Pages = most coherent with email-dev brand identity
Domain: firstnamelastname.com or firstname.dev via Cloudflare Registrar (~$10/yr)
